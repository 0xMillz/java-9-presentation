Breaking changes

A single underscore is no longer a valid variable name

						
						String _ = "foo";

						import sun.misc.Unsafe;

Why Jigsaw?

Why Jigsaw?

  • scale Java SE to small devices
  • improve security and maintainability
  • better application performance
  • easier to construct & maintain libraries/application

JDK after Jigsaw

What is a module?

  • a class is a container of fields and methods
  • a package is a container of classes and interfaces
  • a module is a container of packages

Java Platform Module System

src/module-info.java

            
            module com.dish.ordermanagement {
                requires org.joda.beans;  // this is a module name, not a package name
                requires com.google.guava;

                exports com.dish.ordermanagement.util;  // this is a package name, not a module name
                exports com.dish.ordermanagement.service;
            }

Automatic Modules

  • Real modules
  • Simply place unmodified jar file on module path
  • No changes to JAR file
  • Module name derived from JAR file name
  • Automatically exports all its packages
  • Automatically requires all modules on module path

New Version-String Scheme

Old way:

  • Minor releases containing changes beyond security fixes were multiples of 20
  • Security releases based on the previous minor release were odd numbers incremented by five, or by six....
  • e.g. 1.8.0, 1.8.0_5, 1.8.0_11, 1.8.0_20, 1.8.0_25, 1.8.0_31, 1.8.0_40, 1.8.0_45

New Version String Scheme:

$MAJOR.$MINOR.$SECURITY

  • 1.8.0 ⇒ 8.0.0
  • 1.8.0_05 ⇒ 8.0.1
  • 1.8.0_11 ⇒ 8.0.2
  • 1.8.0_20 ⇒ 8.1.2
  • 1.8.0_25 ⇒ 8.1.3
  • 1.8.0_31 ⇒ 8.1.4
  • 1.8.0_40 ⇒ 8.2.4
  • 1.8.0_45 ⇒ 8.2.5
